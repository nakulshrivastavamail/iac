# Local Kubernetes Cluster with Terraform + Ansible

Provisions a 3-node Kubernetes cluster on KVM/libvirt VMs using Terraform for infrastructure and Ansible for cluster configuration. Designed for WSL2 on Windows with nested virtualization.

## Architecture

- **k8s-control-plane** — 4GB RAM, 2 vCPU, 20GB disk (192.168.122.10)
- **k8s-worker-1** — 2GB RAM, 2 vCPU, 20GB disk (192.168.122.11)
- **k8s-worker-2** — 2GB RAM, 2 vCPU, 20GB disk (192.168.122.12)

All VMs run Ubuntu 24.04 (Noble Numbat) cloud image on a libvirt NAT network (`192.168.122.0/24`).

## Prerequisites

### WSL2 Configuration

Nested virtualization must be enabled. In `%USERPROFILE%\.wslconfig`:

```ini
[wsl2]
nestedVirtualization=true
```

In `/etc/wsl.conf`:

```ini
[boot]
systemd=true

[user]
default=<your-username>

[network]
generateResolvConf = false
```

DNS must be manually configured in `/etc/resolv.conf`:

```
nameserver 8.8.8.8
nameserver 8.8.4.4
```

### Packages

```bash
sudo apt update && sudo apt install -y \
  qemu-kvm libvirt-daemon-system libvirt-clients \
  bridge-utils virtinst mkisofs xsltproc

sudo usermod -aG libvirt $(whoami)
sudo usermod -aG kvm $(whoami)
newgrp libvirt
```

### Terraform

```bash
wget https://releases.hashicorp.com/terraform/1.15.6/terraform_1.15.6_linux_amd64.zip
unzip terraform_1.15.6_linux_amd64.zip
sudo mv terraform /usr/local/bin/
```

### Ansible (in virtual environment)

```bash
python3 -m venv venv
source venv/bin/activate
pip install ansible
```

### SSH Key

```bash
ssh-keygen -t rsa -b 4096
```

The public key at `~/.ssh/id_rsa.pub` is automatically injected into VMs via cloud-init.


## Usage

### Deploy

```bash
cd terraform
terraform init
terraform plan
terraform apply
```

VMs take ~60 seconds after `terraform apply` for cloud-init to complete and SSH to become available.

### Verify

```bash
ssh kube@192.168.122.10
ssh kube@192.168.122.11
ssh kube@192.168.122.12
```

### Destroy

```bash
terraform destroy -auto-approve
```

### Full Rebuild

```bash
terraform destroy -auto-approve && terraform apply
```

## What the Ansible Playbook Does (Bash Equivalent)

These commands show what `k8s-setup.yml` executes on each node.
Do not run these manually — use `ansible-playbook k8s-setup.yml` instead.

### Install prerequisites (line 31)
```bash
sudo apt-get update
sudo apt-get install -y apt-transport-https ca-certificates curl gnupg
```

### Load kernel modules (line 46)
```bash
sudo modprobe overlay
sudo modprobe br_netfilter
```

### Persist modules (line 54)
```bash
cat <<EOF | sudo tee /etc/modules-load.d/k8s.conf
overlay
br_netfilter
EOF
```

### Set sysctl (line 61)
```bash
echo "net.bridge.bridge-nf-call-iptables=1" | sudo tee -a /etc/sysctl.d/k8s.conf
echo "net.bridge.bridge-nf-call-ip6tables=1" | sudo tee -a /etc/sysctl.d/k8s.conf
echo "net.ipv4.ip_forward=1" | sudo tee -a /etc/sysctl.d/k8s.conf
sudo sysctl --system
```

### Disable swap (line 69)
```bash
sudo swapoff -a
sudo sed -i '/swap/d' /etc/fstab
```

### Install containerd (line 77)
```bash
sudo apt-get install -y containerd
```

### Configure containerd (line 83-94)
```bash
sudo mkdir -p /etc/containerd
sudo containerd config default | sudo tee /etc/containerd/config.toml
sudo sed -i 's/SystemdCgroup = false/SystemdCgroup = true/' /etc/containerd/config.toml
sudo systemctl restart containerd
sudo systemctl enable containerd
```

### Add K8s repo (line 103-114)
```bash
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://pkgs.k8s.io/core:/stable:/v1.35/deb/Release.key \
  | sudo gpg --dearmor -o /etc/apt/keyrings/kubernetes-apt-keyring.gpg
echo "deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] \
  https://pkgs.k8s.io/core:/stable:/v1.35/deb/ /" \
  | sudo tee /etc/apt/sources.list.d/kubernetes.list
```

### Install K8s packages (line 119)
```bash
sudo apt-get update
sudo apt-get install -y kubelet kubeadm kubectl
sudo apt-mark hold kubelet kubeadm kubectl
sudo systemctl enable kubelet
```

### --- Control plane only ---

### Init cluster (line 146)
```bash
sudo kubeadm init \
  --apiserver-advertise-address=192.168.122.10 \
  --pod-network-cidr=10.244.0.0/16 \
```


### Install Flannel (line 166)
```bash
kubectl apply -f https://github.com/flannel-io/flannel/releases/latest/download/kube-flannel.yml
```

### Get join command (line 172)
```bash
kubeadm token create --print-join-command
```

### --- Workers only ---

### Join cluster (line 188)
```bash
sudo kubeadm join 192.168.122.10:6443 \
  --token abc123 \
  --discovery-token-ca-cert-hash sha256:xyz \
  --node-name=k8s-worker-1
```

```bash

scp kube@192.168.122.10:$HOME/.kube/config $HOME/.kube/config
```

## How It Works

**Terraform** creates the infrastructure: a libvirt storage pool, downloads the Ubuntu 24.04 cloud image, creates thin-clone disks for each node, builds cloud-init ISOs with user/network configuration, and defines KVM domains (VMs).

**Cloud-init** runs on first boot inside each VM: sets the hostname, creates the `kube` user with SSH key access, configures static IPs, disables swap, and loads kernel modules required by Kubernetes.

**Ansible** inventory is auto-generated by Terraform from the same node definitions in `variables.tf`, ensuring IPs are defined in one place.

## Known Issues (WSL2-Specific)

- **DNS in VMs**: libvirt's default NAT DNS forwarding may not work in WSL2. VMs use `8.8.8.8` / `8.8.4.4` directly instead of `192.168.122.1`.
- **Package updates in cloud-init**: `package_update: true` can hang cloud-init indefinitely if internet access from VMs is unreliable. Package installation is handled by Ansible instead.
- **Serial console**: `virsh console` requires kernel boot parameters (`console=ttyS0`) which aren't set by default on the cloud image. Use SSH for access.
- **libguestfs**: `virt-cat` / `guestmount` tools fail in WSL2 due to supermin issues. Use `qemu-nbd` to mount VM disks for debugging.

## Customization

Node count, IPs, memory, CPU, and disk size are configured in `variables.tf`:

```hcl
variable "nodes" {
  default = {
    "k8s-control-plane" = { ip = "192.168.122.10", memory = 4096, vcpu = 2, disk_gb = 20 }
    "k8s-worker-1"      = { ip = "192.168.122.11", memory = 2048, vcpu = 2, disk_gb = 20 }
    "k8s-worker-2"      = { ip = "192.168.122.12", memory = 2048, vcpu = 2, disk_gb = 20 }
  }
}
```
